LinkVault Q8 iOS package

Configured bundle identifiers:
- App: com.linkvaultq8.app
- Share Extension: com.linkvaultq8.app.ShareExtension

Important fix in this package:
- The Share Extension product name is forced to LinkVaultShareExtension.appex.
- Stale/empty .appex references are cleaned from the Xcode project.
- App Group entitlements are enabled by default and unified on: group.com.linkvaultq8.shared.
- In Apple Developer, enable the same App Group for both identifiers before building:
  App: com.linkvaultq8.app
  Share Extension: com.linkvaultq8.app.ShareExtension
- Codemagic keeps USE_APP_GROUPS=true and APP_GROUP_ID=group.com.linkvaultq8.shared so the generated entitlements match the Swift files.

Runtime Share Extension UI fix:
- Share Extension now shows a small native confirmation screen with Open and Close buttons.
- It no longer tries to auto-open the app immediately on selection, which prevents host apps from appearing frozen.
- User can always close the share extension safely.


Share Extension open fix:
- Added primary URL scheme linkvaultq8:// and kept linkvault:// as fallback.
- Share Extension now opens the main app using extensionContext.open after the URL is saved.

Applied share save reliability update:
- Share Extension now opens linkvaultq8://share after saving, so the app receives the link immediately.
- PendingSharePlugin no longer clears App Group data during getPendingShare(); the web app clears it only after the Add Link modal is opened.
- Web app clears pending share after a successful deep-link or App Group consume and de-duplicates repeated delivery.
